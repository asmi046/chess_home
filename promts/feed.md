Сделай модель с имененм OwerFlatInformation, к нему создай миграцию и сидер.

Поля модели:

- Тип квартиры - строковое обязательное длинна 50 символов
- Галерея - json - поле nullable
- Город - строковое nullable
- Страна - строковое nullable

Сидер собери на основе папки public/flat_data по следующим правилам:

- Тип квартиры - слаг из имени вложенной директории
- Галерея - входим в директорию и формируем массив в котором имя вложенной в папку картинки и тайтл который пока пустой
- Адрес - проставляем константу Республика Крым, г. Ялта, пгт. Гурзуф, в районе Гурзуфской средней школы
- Город - проставляем константу Гурзуф
- Страна - проставляем константу Россия

Далее создай миграцию которая добавляет в таблицу flats поле с типом квартиры для связи с таблицей созданной для модели OwerFlatInformation.

## Создание фида в формате YRL

Фид будет строится по таблице flats.

Для решения этой задачи создай контроллер который будет реализовывать метод для вывода фида назови его FeedController. Метод принимает парметр с id секции квартиры которой нужно вывести в фид. По умолчанию параметр = null что говорит о том что в выборку попадают все квартиры из таблицы flats у которых заполнено дополнительные данные. Если параметр передан то выбираются квартиры у котолрых section_id = переданному параметру и заполненны дополнительные данные.

Создай соответствующий роут и размести его в файле routes/asmi_all.php. Роут должен быть публичным. В роут должен передаваться необязательный параметр с id секции квартиры которой выводятся в фид

Создай соответствующую view где будект формироваться фид.
